---
description: Structured brainstorming process before coding.
---

1. **Requirements**
   - State the User Goal clearly.
   - List constraints (performance, existing architecture).

2. **Exploration**
   - Check existing components/libraries to reuse.
   - Research patterns (e.g., "How do other apps handle this?").

3. **Ideation (Brainstorming)**
   - Propose 2-3 approaches.
     - *Option A:* (e.g., Quick & Dirty / Client-side)
     - *Option B:* (e.g., Robust / Server-side)
   - List Pros/Cons for each.

4. **Decision**
   - Select the best approach with the User.
   - Define the "Definition of Done".

5. **Plan**
   - Generate/Update `implementation_plan.md`.
   - Break down into tasks.
